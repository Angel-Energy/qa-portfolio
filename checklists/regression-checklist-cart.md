# Regression Checklist — корзина интернет-магазина

## Цель
Проверить, что изменения в функциональности корзины
не повлияли на уже работавшие сценарии.

## Область проверки
- добавление товаров
- изменение количества
- удаление товаров
- сохранение корзины
- расчёт стоимости
- скидки и акции
- переход к оформлению заказа

---

## Checklist

| ID | Проверка | Результат |
|----|----------|-----------|
| REG-001 | Товар добавляется в корзину из карточки товара | Pass / Fail |
| REG-002 | Товар добавляется в корзину из списка товаров | Pass / Fail |
| REG-003 | Один и тот же товар можно добавить несколько раз | Pass / Fail |
| REG-004 | Количество товара увеличивается кнопкой "+" | Pass / Fail |
| REG-005 | Количество товара уменьшается кнопкой "-" | Pass / Fail |
| REG-006 | Товар удаляется из корзины | Pass / Fail |
| REG-007 | Стоимость пересчитывается после изменения количества | Pass / Fail |
| REG-008 | Итоговая сумма считается корректно | Pass / Fail |
| REG-009 | Скидка применяется корректно | Pass / Fail |
| REG-010 | Товары остаются в корзине после перезапуска приложения / обновления страницы | Pass / Fail |
| REG-011 | Корзина корректно работает с несколькими разными товарами | Pass / Fail |
| REG-012 | Переход к оформлению заказа доступен | Pass / Fail |
| REG-013 | После оформления заказа корзина очищается корректно | Pass / Fail |
| REG-014 | Неактуальный или недоступный товар корректно обрабатывается системой | Pass / Fail |
| REG-015 | Ошибки интерфейса в корзине отсутствуют | Pass / Fail |

---

## Критерий прохождения regression-теста
Регрессионная проверка считается успешной,
если после изменений корзина сохраняет
корректную работу всех основных сценариев:
добавление, изменение количества, удаление,
расчёт суммы, применение скидки и переход к заказу.
